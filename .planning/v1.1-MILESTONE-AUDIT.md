## Cross-Phase Integration

### Critical Wiring Gaps

**1. DispatchSettings not in ZerothSettings** (OPS-04, OPS-05)
- Phase 16 created `DispatchSettings` with `arq_enabled`, `shutdown_timeout`, `poll_interval`
- Not merged into `src/zeroth/config/settings.py` ZerothSettings class
- docker-compose.yml references `ZEROTH_DISPATCH__ARQ_ENABLED` which has no target

**2. ARQ bootstrap wiring missing** (OPS-05)
- `arq_pool` field not on ServiceBootstrap dataclass
- `create_arq_pool` never called in `bootstrap_service()`
- Lifespan code handles gracefully via `getattr` but feature is dead

**3. InstrumentedProviderAdapter not wired** (ECON-01, ECON-02)
- Adapter exists in `econ/adapter.py`, wraps any ProviderAdapter
- No code in bootstrap or orchestrator instantiates it
- Cost events never emitted in default deployment

**4. Memory factory uses stub settings** (MEM-01..06)
- `bootstrap.py` passes `_BootstrapMemorySettings()` instead of real `ZerothSettings`
- External memory backends (Redis, pgvector, ChromaDB, ES) never registered
- Only in-memory connectors work in production

**5. cost_api double /v1/ prefix** (ECON-04, DEP-02)
- Route decorators have hardcoded `/v1/` paths
- Mounted on v1_router with prefix=/v1/ creates `/v1/v1/...`

## Recommended Fix Plan

The 5 integration gaps share a common root cause: **worktree merge gaps**. A single gap-closure phase can fix all of them:

1. **Add DispatchSettings to ZerothSettings** — merge from Phase 16 worktree
2. **Wire arq_pool in ServiceBootstrap** — add field, create pool in bootstrap_service()
3. **Wire InstrumentedProviderAdapter in bootstrap** — wrap agent providers
4. **Pass real ZerothSettings to memory factory** — replace stub
5. **Fix cost_api route paths** — remove hardcoded /v1/ prefix
6. **Update stale REQUIREMENTS.md markers** — ECON-03, MEM-06, OPS-04, OPS-05

Estimated scope: 1 plan, ~6 tasks, all in shared files (settings.py, bootstrap.py, cost_api.py, REQUIREMENTS.md).
