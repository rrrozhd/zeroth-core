## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| orchestrator -> http_client | Orchestrator passes effective_capabilities to gate requests |
| http_client -> SecretProvider | Runtime secret resolution for auth headers |
| http_client -> audit accumulator | Call records with sensitive data redacted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-07 | Elevation of Privilege | Bootstrap wiring | mitigate | HTTP client receives effective_capabilities from orchestrator enforcement context; cannot bypass PolicyGuard evaluation that already happened at node level |
| T-35-08 | Information Disclosure | Audit record attachment | mitigate | HttpCallRecord uses redact_url() for URL; auth headers never included in records; only url, method, status, latency, size logged (per D-11) |
| T-35-09 | Information Disclosure | SecretProvider env resolution | mitigate | EnvSecretProvider reads os.environ at runtime; secret values exist only in function-scope variables during auth header injection; never persisted in client state or config models |
| T-35-10 | Tampering | drain_call_records | accept | Records are informational for audit; if tampered before attachment to NodeAuditRecord, audit chain verification (digest-based) will detect inconsistencies. Low risk: internal API only. |
</threat_model>
