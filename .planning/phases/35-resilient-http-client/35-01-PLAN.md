<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> external API | Untrusted network; responses may be malicious, slow, or missing |
| caller -> ResilientHttpClient | Caller provides URL, headers, body; URL may target internal services (SSRF) |
| config -> secrets | SecretProvider resolves sensitive tokens that must not leak to audit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Information Disclosure | HttpCallRecord audit | mitigate | redact_url() strips query params; auth headers never logged; only URL path, method, status, latency, size in audit record (per D-11) |
| T-35-02 | Spoofing | Auth header injection | mitigate | SecretProvider resolves at runtime; secrets never stored in EndpointConfig or HttpClientSettings models; resolved headers exist only in function scope |
| T-35-03 | Denial of Service | Retry storm amplification | mitigate | Circuit breaker stops retries after threshold (per D-06); jitter in backoff formula prevents synchronized retries (per D-05); rate limiter caps requests per endpoint (per D-12) |
| T-35-04 | Denial of Service | Connection exhaustion | mitigate | httpx.Limits caps max_connections and max_keepalive_connections (per D-08); rate limiter provides request-level throttling |
| T-35-05 | Elevation of Privilege | Capability bypass | mitigate | effective_capabilities set checked before every request; missing capability raises HttpClientError; NETWORK_READ/WRITE/EXTERNAL_API_CALL required per method (per D-10) |
| T-35-06 | Spoofing | SSRF via arbitrary URL | accept | Capability gating provides defense-in-depth (only authorized nodes can make calls); full URL allowlisting is a future enhancement. Risk accepted: the caller is platform code (agent tools, execution units), not external user input. |
</threat_model>
