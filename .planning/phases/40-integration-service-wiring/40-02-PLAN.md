  <action>
    **Create `src/zeroth/core/service/artifact_api.py`:**

    Follow the `webhook_api.py` pattern exactly. Define `register_artifact_routes(app: FastAPI | APIRouter) -> None`. Create:

    - `GET /artifacts/{artifact_id}`: Access `request.app.state.bootstrap.artifact_store`. If None, raise `HTTPException(503, "artifact store not configured")`. Call `await store.retrieve(artifact_id)`. On `ArtifactNotFoundError`, raise `HTTPException(404)`. Return `Response(content=data, media_type="application/octet-stream")`. Require `Permission.RUN_READ` (artifacts are tied to runs).

    - Helper `_artifact_store(request)` that extracts and validates the store from bootstrap, raising 503 if None.

    No write/delete endpoints for artifacts (per D-04, artifact store is for internal payload externalization; retrieval is the external API surface).

    **Create `src/zeroth/core/service/template_api.py`:**

    Follow the same pattern. Define `register_template_routes(app: FastAPI | APIRouter) -> None`. Create:

    - Pydantic request/response models:
      - `CreateTemplateRequest(BaseModel)`: name (str), version (int, default 1), body (str), variables (list[str], default []), description (str, default ""). Use `ConfigDict(extra="forbid")`.
      - `TemplateResponse(BaseModel)`: name, version, body, variables, description.
      - `TemplateListResponse(BaseModel)`: templates (list[TemplateResponse]).

    - `GET /templates`: List all templates from registry. Require `Permission.RUN_READ`.
    - `POST /templates`: Register a new template. On `TemplateVersionExistsError`, raise `HTTPException(409)`. Return 201 + created template. Require `Permission.ADMIN`.
    - `GET /templates/{name}`: Get latest version. Optional query param `version: int | None = None`. On `TemplateNotFoundError`, raise `HTTPException(404)`. Require `Permission.RUN_READ`.
    - `DELETE /templates/{name}/{version}`: Remove a template version. Since TemplateRegistry has no delete method, implement a simple `_templates` dict pop in the route. If not found, raise 404. Return 204 (no content). Require `Permission.ADMIN`.

    Note: Check if TemplateRegistry has a `delete` or `remove` method. If not, the DELETE endpoint should return 501 (not implemented) or skip the delete endpoint entirely and document it. Read the registry source first.

    Helper `_template_registry(request)` to extract from bootstrap, raising 503 if None.

    **Create `tests/service/test_artifact_api.py`:**

    Use `TestClient` + `bootstrap_app` pattern from existing service tests. Create a `FilesystemArtifactStore(tmp_path)` and set it on the bootstrap. Test:
    - Store an artifact, GET it by key -> 200 + correct bytes
    - GET unknown key -> 404
    - Bootstrap with artifact_store=None -> 503

    **Create `tests/service/test_template_api.py`:**

    Use `TestClient` + `bootstrap_app` pattern. Create a `TemplateRegistry()` and set on bootstrap. Test:
    - POST a template -> 201
    - GET /templates -> list includes the template
    - GET /templates/{name} -> returns template
    - GET /templates/{name}?version=1 -> returns specific version
    - GET /templates/nonexistent -> 404
    - POST duplicate name+version -> 409

    Use `admin_headers()` for write operations, `operator_headers()` for read operations.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> artifact API | Untrusted client requests artifact data by key; must validate key exists |
| client -> template API | Untrusted client submits template bodies; must validate input shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-04 | Information Disclosure | artifact_api.py GET | mitigate | Require RUN_READ permission; artifact keys are opaque UUIDs so enumeration is infeasible; return 404 not 403 for missing keys |
| T-40-05 | Spoofing | template_api.py POST | mitigate | Require ADMIN permission for template creation/deletion; read-only access for RUN_READ |
| T-40-06 | Tampering | template_api.py POST | mitigate | PromptTemplate is immutable once registered (TemplateVersionExistsError on duplicate); Pydantic ConfigDict(extra="forbid") rejects unknown fields |
| T-40-07 | Denial of Service | template_api.py POST body | accept | Template body size is bounded by FastAPI default request body limit (1MB); no additional limit needed at ASVS L1 |
</threat_model>
