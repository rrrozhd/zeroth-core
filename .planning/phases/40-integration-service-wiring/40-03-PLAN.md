    **Per D-07:** Update existing documentation to reference new v4.0 capabilities. No new doc pages (docs site is in zeroth-studio repo). Only update files that already exist.

    **Step 1: Update `docs/reference/http-api.md`.**

    Add a section (or update existing) mentioning the new artifact and template endpoints:

    - `GET /v1/artifacts/{artifact_id}` -- Retrieve an externalized artifact by key. Returns raw binary content. Requires `run:read` permission. Returns 404 if artifact not found, 503 if artifact store not configured.

    - `GET /v1/templates` -- List all registered prompt templates. Requires `run:read` permission.
    - `POST /v1/templates` -- Register a new prompt template. Requires `admin` permission. Returns 201 on success, 409 if version already exists.
    - `GET /v1/templates/{name}` -- Get a template by name (latest version). Optional `?version=N` query param. Requires `run:read` permission. Returns 404 if not found.
    - `DELETE /v1/templates/{name}/{version}` -- Remove a template version. Requires `admin` permission. Returns 204 on success.

    **Step 2: Update parallel execution docs.**

    If `docs/concepts/parallel.md` exists, add a "Known Limitations" or "Restrictions" subsection noting:

    > SubgraphNode cannot be used as a downstream node inside parallel branches. If a parallel fan-out source node has a SubgraphNode as a downstream node, the fan-out will be rejected with a validation error. This restriction exists because subgraph execution requires the full `_drive()` loop which is not available inside branch coroutines. Subgraph composition and parallel execution work independently but cannot be nested.

    **Step 3: Update subgraph composition docs.**

    If `docs/concepts/subgraph.md` exists (or similar), add a note in limitations:

    > Subgraph nodes inside parallel branches are not currently supported. Subgraph composition works at the top-level `_drive()` loop and within other subgraphs (nested subgraphs are supported), but not inside parallel fan-out branches.

    **Step 4: Update artifact and template docs.**

    If `docs/concepts/artifacts.md` and `docs/concepts/templates.md` exist, add brief mentions that REST API endpoints are now available for retrieval/management:

    - Artifacts: "Artifacts can be retrieved via the HTTP API using `GET /v1/artifacts/{artifact_id}`."
    - Templates: "Templates can be managed via the HTTP API using the `/v1/templates` endpoints (list, create, get, delete)."

    **Important:** Only modify files that exist. If a file does not exist, skip that update. Use `ls docs/concepts/` to find exact filenames before editing.
