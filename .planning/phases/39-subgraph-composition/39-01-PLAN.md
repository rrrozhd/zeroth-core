<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| graph_ref input | Untrusted string from graph definition -- could reference non-existent or unauthorized graphs |
| serialized_graph deserialization | Stored JSON parsed into Graph objects -- malformed data could cause crashes |
| parent_run_id | New field on Run -- must not allow run linkage across tenant boundaries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Spoofing | SubgraphResolver | mitigate | graph_ref resolved via DeploymentService which enforces deployment_ref lookup -- only published deployments are reachable |
| T-39-02 | Tampering | namespace_subgraph | mitigate | Returns model_copy (immutable pattern) -- original graph never modified; prefix is deterministic from graph_ref+depth |
| T-39-03 | Information Disclosure | parent_run_id | accept | parent_run_id is internal metadata; Run objects are scoped by tenant_id in all repository queries |
| T-39-04 | Denial of Service | SubgraphNodeData.max_depth | mitigate | max_depth constrained to ge=1, le=10 via Pydantic Field validator; prevents unbounded recursion at model level |
| T-39-05 | Elevation of Privilege | merge_governance | mitigate | Parent policies prepended to subgraph policies; PolicyGuard intersection semantics ensure subgraph can only restrict, never relax parent capabilities |
</threat_model>
