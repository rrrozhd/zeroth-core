```python
@dataclass(slots=True)
class RuntimeOrchestrator:
    subgraph_executor: Any | None = None  # Phase 39: SubgraphExecutor

    async def resume_graph(self, graph: Graph, run_id: str) -> Run:
        """Resume a paused run (e.g., after approval)."""
        run = await self.run_repository.get(run_id)
        if run is None:
            raise OrchestratorError(f"run {run_id!r} not found")
        run.status = RunStatus.RUNNING
        run.touch()
        run = await self.run_repository.put(run)
        return await self._drive(graph, run)
```

Approval propagation flow (from RESEARCH.md):
```
Parent _drive() -> SubgraphNode -> execute() -> child _drive()
  -> child hits HumanApprovalNode -> child status = WAITING_APPROVAL
  -> execute() returns child_run with WAITING_APPROVAL
  -> parent detects WAITING_APPROVAL
  -> parent stores pending_subgraph = {child_run_id, node_id, graph_ref}
  -> parent re-queues SubgraphNode at front of pending_node_ids
  -> parent status = WAITING_APPROVAL
  -> parent _drive() returns

  ... human resolves approval ...

  -> resume_graph(parent_run_id) called
  -> parent _drive() pops SubgraphNode (re-queued)
  -> parent detects pending_subgraph metadata
  -> parent calls resume_graph(subgraph, child_run_id) instead of execute()
  -> child _drive() resumes, completes
  -> parent gets child.final_output, clears pending_subgraph, continues
```

    ```python
    pending_subgraph = run.metadata.get("pending_subgraph")
    if pending_subgraph and pending_subgraph.get("node_id") == node_id:
        # Resuming after approval -- continue the existing child run
        child_run_id = pending_subgraph["child_run_id"]
        graph_ref = pending_subgraph["graph_ref"]
        version = pending_subgraph.get("version")

        # Re-resolve, re-namespace, re-merge governance (Graph objects are not persisted in metadata)
        try:
            subgraph, _ = await self.subgraph_executor.resolver.resolve(graph_ref, version)
        except (SubgraphResolutionError,) as exc:
            return await self._fail_run(run, "subgraph_resume_failed", str(exc))

        depth = run.metadata.get("subgraph_depth", 0) + 1
        subgraph = namespace_subgraph(subgraph, graph_ref, depth)
        subgraph = merge_governance(graph, subgraph)

        # Resume the child run (not create a new one)
        child_run = await self.resume_graph(subgraph, child_run_id)

        if child_run.status == RunStatus.WAITING_APPROVAL:
            # Still waiting (nested approval or another approval in subgraph)
            # Stay paused -- pending_subgraph metadata already correct
            return run  # Already in WAITING_APPROVAL

        # Child completed -- clear pending state, use output
        del run.metadata["pending_subgraph"]
        run.status = RunStatus.RUNNING
        output_data = child_run.final_output or {}
        if not isinstance(output_data, dict):
            output_data = {"result": output_data}

        audit_record = {
            "subgraph_run_id": child_run.run_id,
            "subgraph_graph_ref": graph_ref,
            "subgraph_status": child_run.status.value,
            "subgraph_resumed": True,
        }

        # Continue normal post-node flow
        await self._record_history(run, node, node_id, input_payload, output_data, audit_record)
        self._increment_node_visit(run, node_id)
        next_node_ids = self._plan_next_nodes(graph, run, node_id, output_data)
        self._queue_next_nodes(graph, run, node_id, output_data, next_node_ids)
        run.metadata["last_output"] = output_data
        run.touch()
        persisted = await self.run_repository.put(run)
        await self.run_repository.write_checkpoint(persisted)
        await self._refresh_artifact_ttls(persisted)
        continue
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| approval resolution -> parent resume | An approved child run must correctly trigger parent resumption without allowing unauthorized state transitions |
| pending_subgraph metadata | Stored in run.metadata -- must not be tamper-vulnerable to cause incorrect child resumption |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-11 | Tampering | pending_subgraph metadata | mitigate | Metadata validated on resume: child_run_id must exist, node_id must match current node, graph_ref must resolve. Invalid metadata causes run failure, not silent corruption. |
| T-39-12 | Repudiation | Approval resolution chain | mitigate | Audit records track subgraph_resumed=True flag and child_run_id, creating an unbroken chain from approval resolution through child completion to parent continuation |
| T-39-13 | Denial of Service | Nested approvals | accept | Each nesting level adds one pause/resume cycle. With max_depth=10, at most 10 approval pauses can chain. This is bounded and each pause is a human-speed operation. |
| T-39-14 | Information Disclosure | Re-resolution on resume | accept | The subgraph is re-resolved from DeploymentService on resume. If the deployment was updated between pause and resume, the version pin prevents drift. If no version pin, the latest deployment is used (by design). |
</threat_model>
