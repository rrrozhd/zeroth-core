    1. Create `src/zeroth/core/subgraph/executor.py`:

       ```python
       @dataclass(slots=True)
       class SubgraphExecutor:
           """Executes subgraph nodes by creating child Runs and recursively driving them."""
           resolver: SubgraphResolver

           async def execute(
               self,
               orchestrator: RuntimeOrchestrator,
               parent_graph: Graph,
               parent_run: Run,
               node: SubgraphNode,
               node_id: str,
               input_payload: dict[str, Any],
           ) -> Run:
       ```

       The `execute()` method does the following in order:
       a. **Depth check** (per D-04): Read current depth from `parent_run.metadata.get("subgraph_depth", 0)`. New depth = current + 1. If new depth > `node.subgraph.max_depth`, raise `SubgraphDepthLimitError(f"subgraph depth {new_depth} exceeds max_depth {node.subgraph.max_depth} for graph_ref '{node.subgraph.graph_ref}'")`

       b. **Cycle detection** (per research Pitfall 2): Read visited set from `parent_run.metadata.get("visited_subgraph_refs", [])`. If `node.subgraph.graph_ref` is in the set, raise `SubgraphCycleError(f"circular subgraph reference detected: {node.subgraph.graph_ref}")`. Add current graph_ref to the visited set for the child.

       c. **Resolve** (per D-02): Call `self.resolver.resolve(node.subgraph.graph_ref, node.subgraph.version)` to get `(subgraph, deployment)`

       d. **Namespace** (per D-10): Call `namespace_subgraph(subgraph, node.subgraph.graph_ref, new_depth)` to prefix all node/edge IDs

       e. **Governance merge** (per D-06): Call `merge_governance(parent_graph, namespaced_subgraph)` to prepend parent policies

       f. **Create child Run** (per D-05, D-08):
          - thread_id = parent_run.thread_id if thread_participation == "inherit" else "" (empty triggers auto-gen)
          - Create Run with:
            - graph_version_ref from subgraph
            - deployment_ref = node.subgraph.graph_ref
            - tenant_id = parent_run.tenant_id
            - workspace_id = parent_run.workspace_id
            - thread_id as computed above
            - parent_run_id = parent_run.run_id
            - pending_node_ids = [orchestrator._entry_step(merged_subgraph)]
            - metadata with subgraph_depth, parent_run_id, parent_node_id, visited_subgraph_refs (current set + graph_ref)
            - initial_input stored in metadata for the entry step payload
          - Persist via orchestrator.run_repository.create()
          - Set status = RUNNING, touch(), persist via put()

       g. **Execute** (per D-03): Call `orchestrator._drive(merged_subgraph, child_run)` -- the recursive invocation
          - Wrap in try/except for unexpected errors -> SubgraphExecutionError
          - Return the resulting child_run

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| orchestrator -> subgraph executor | Parent graph context enters SubgraphExecutor -- must not leak parent state to isolated subgraphs |
| subgraph child Run | New Run created with potentially different tenant -- must inherit parent's tenant_id |
| recursive _drive() | Same orchestrator instance re-enters _drive() -- must be stateless between calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-06 | Denial of Service | SubgraphExecutor.execute() | mitigate | Depth counter checked BEFORE resolution/execution; max_depth capped at 10 by model validator; cycle detection prevents infinite loops even within depth limit |
| T-39-07 | Tampering | Child Run creation | mitigate | Child Run inherits tenant_id and workspace_id from parent -- no cross-tenant escalation possible; parent_run_id creates traceable lineage |
| T-39-08 | Information Disclosure | Thread participation "inherit" | accept | Documented behavior per D-08: "inherit" means full bidirectional thread sharing. Users who want isolation must use "isolated" mode. |
| T-39-09 | Elevation of Privilege | Recursive _drive() | mitigate | Governance merge (parent-ceiling) applied before _drive() call ensures subgraph cannot relax parent policies; the merged graph is a copy, not a reference to the original |
| T-39-10 | Spoofing | Bootstrap wiring | mitigate | SubgraphExecutor wired with same DeploymentService instance used for parent graph -- no separate trust path; resolution uses existing deployment_ref lookup with version validation |
</threat_model>
