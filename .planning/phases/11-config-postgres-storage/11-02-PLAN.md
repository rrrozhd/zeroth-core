Add a new function to run Alembic migrations at startup:
```python
def run_migrations(database_url: str) -> None:
    """Run Alembic migrations against the given database URL."""
    from alembic.config import Config
    from alembic import command
    import importlib.resources

    alembic_cfg = Config()
    migrations_dir = str(importlib.resources.files("zeroth.migrations"))
    alembic_cfg.set_main_option("script_location", migrations_dir)
    alembic_cfg.set_main_option("sqlalchemy.url", database_url)
    command.upgrade(alembic_cfg, "head")
```

This is a SYNC function because Alembic uses SQLAlchemy's sync engine internally. Call it before creating the async database connection.
