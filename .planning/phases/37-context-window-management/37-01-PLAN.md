       ```python
       class ContextWindowTracker:
           def __init__(self, settings: ContextWindowSettings, strategy: "CompactionStrategy") -> None:
               self.settings = settings
               self.strategy = strategy
               self._accumulated_tokens: int = 0
               self._compaction_count: int = 0

           def count_tokens(self, messages: list[Any], model_name: str) -> int:
               if not messages:
                   return 0
               try:
                   return litellm.token_counter(model=model_name, messages=self._normalize_messages(messages))
               except Exception as exc:
                   msg = f"token counting failed for model {model_name}: {exc}"
                   raise TokenCountError(msg) from exc

           def needs_compaction(self, token_count: int) -> bool:
               if self.settings.max_context_tokens <= 0:
                   return False
               ratio = token_count / self.settings.max_context_tokens
               return ratio >= self.settings.summary_trigger_ratio

           async def maybe_compact(self, messages: list[Any], model_name: str) -> tuple[list[Any], CompactionResult | None]:
               token_count = self.count_tokens(messages, model_name)
               self._accumulated_tokens = token_count
               if not self.needs_compaction(token_count):
                   return messages, None
               result = await self.strategy.compact(messages, settings=self.settings, model_name=model_name)
               self._accumulated_tokens = result.tokens_after
               self._compaction_count += 1
               return list(result.messages), result

           @property
           def state(self) -> CompactionState:
               return CompactionState(accumulated_tokens=self._accumulated_tokens, max_tokens=self.settings.max_context_tokens, compaction_count=self._compaction_count, last_compaction_strategy=None)
       ```
       The `_normalize_messages` helper converts PromptMessage pydantic objects to dicts for litellm compatibility (check if message has `model_dump`, call it; otherwise pass through).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| litellm.token_counter | External library call with untrusted message content |
| LLM summarization call | ProviderAdapter invocation from compaction (not user-initiated) |
| Message content manipulation | Compaction modifies message list contents |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01 | Tampering | strategies.py | mitigate | Compaction returns NEW message lists -- never mutates original messages in-place (prevents corrupting audit records or other references to the same list). Verified by test assertions comparing original list identity. |
| T-37-02 | Information Disclosure | strategies.py | mitigate | ObservationMaskingStrategy replaces content with generic placeholder -- no original content leaks into the placeholder text. Token count is approximate, not content-derived. |
| T-37-03 | Denial of Service | tracker.py | mitigate | TokenCountError wraps litellm exceptions so a counting failure does not crash the agent runner. Caller (runner in Plan 02) can catch and skip compaction gracefully. |
| T-37-04 | Elevation of Privilege | strategies.py | mitigate | LLMSummarizationStrategy uses the same ProviderAdapter as the agent -- inherits existing cost instrumentation, budget enforcement, and audit. No separate unmonitored LLM path. |
| T-37-05 | Spoofing | models.py | accept | ContextWindowSettings is per-node config from graph author (trusted). No untrusted user input reaches settings at runtime. |
</threat_model>
