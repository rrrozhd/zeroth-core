## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Graph author -> AgentNodeData | Per-node context_window settings set by graph author (trusted) |
| Runner -> ThreadStateStore | Compacted messages persisted and restored from thread state |
| Orchestrator -> Runner | Tracker injection at dispatch boundary |
| Compaction -> Audit | Compaction metadata flows into audit records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-06 | Tampering | runner.py | mitigate | compacted_messages in thread state are written by the runner itself (trusted path). On restore, messages are used as-is -- same trust level as existing thread_state restoration. No untrusted external input reaches compacted_messages. |
| T-37-07 | Information Disclosure | runner.py | mitigate | archived_messages (D-08) stored in thread state alongside compacted messages. Thread state is scoped to thread_id and tenant_id via ThreadRepository -- existing tenant isolation applies. Archived messages are NOT included in audit records (only compaction metadata is). |
| T-37-08 | Denial of Service | runtime.py | mitigate | ContextWindowTracker creation is lightweight (no I/O). Compaction itself is bounded by message list size. LLM summarization strategy uses the same provider with existing timeout and budget enforcement. A compaction failure is caught and does not crash the run (runner continues with original messages). |
| T-37-09 | Repudiation | runner.py | mitigate | Compaction metadata (strategy, tokens before/after, message counts) recorded in audit record's execution_metadata. Auditor can see that compaction occurred and what changed. |
| T-37-10 | Spoofing | runtime.py | accept | Strategy selection based on string from AgentNodeData (graph author config). Invalid strategy name falls through to default (observation_masking). No code injection possible -- strategy classes are hardcoded in the orchestrator switch. |
</threat_model>
