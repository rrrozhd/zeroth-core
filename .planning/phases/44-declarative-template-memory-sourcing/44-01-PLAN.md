<objective>
Add declarative memory sourcing to agent prompt templates. An `AgentNodeData` node can now list `template_memory_bindings` that the orchestrator resolves at dispatch time and injects into the `{{ memory.* }}` namespace before Jinja2 template rendering. This eliminates the need for dedicated memory-read nodes before every templated agent step.

Output: `TemplateMemoryBinding` model, `_resolve_template_memory` orchestrator method, `MemoryBindingResolutionError`, and 13 tests — all green, zero regressions.
</objective>

## Task List

### Task 1 — Data model

**Files:** `src/zeroth/core/graph/models.py`, `src/zeroth/core/graph/__init__.py`

Add `TemplateMemoryBinding` Pydantic model before `AgentNodeData`:

```python
class TemplateMemoryBinding(BaseModel):
    as_name: str
    connector_instance_id: str
    access_mode: Literal["get", "scan"] = "get"
    key: str | None = None
    key_prefix: str | None = None
    default: Any = None
    max_items: int | None = Field(default=None, ge=1)
    scope: Literal["run", "thread", "shared"] = "run"

    @model_validator(mode="after")
    def _validate_mode_fields(self) -> TemplateMemoryBinding:
        if self.access_mode == "get" and self.key is None:
            raise ValueError("key is required when access_mode is 'get'")
        return self
```

Add `template_memory_bindings: list[TemplateMemoryBinding] = Field(default_factory=list)` to `AgentNodeData`.

Export `TemplateMemoryBinding` from `zeroth.core.graph`.

### Task 2 — Orchestrator wiring

**File:** `src/zeroth/core/orchestrator/runtime.py`

1. Add `import re` at module level.
2. Add `MemoryBindingResolutionError(OrchestratorError)` class.
3. Add `_KEY_PLACEHOLDER_RE = re.compile(r"\{(input|state|run)\.([^}]+)\}")` module-level constant.
4. Add `_substitute_tmb_key(key, *, input_payload, state, run_id) -> str` module-level helper using regex sub.
5. In `_dispatch_node`:
   - Move `thread_id = await self._resolve_thread(node, run)` to **before** the template `if` block.
   - Add `tmb_audit_records: list[dict[str, Any]] = []` immediately after.
   - Inside the template block, before building `render_vars`: call `_resolve_template_memory`, extend `tmb_audit_records`.
   - Replace `"memory": {}` with `"memory": _memory_ns`.
   - After the `finally` block: `if tmb_audit_records: audit_record["execution_metadata"]["template_memory_bindings"] = tmb_audit_records`.
6. Add `_resolve_template_memory(node, run, thread_id, input_payload)` method:
   - Early-return `({}, [])` when `not bindings or self.memory_resolver is None`.
   - Resolve only the connector refs used by the bindings via `self.memory_resolver.resolve(refs_needed, ...)`.
   - For `"get"`: call `connector.read(resolved_key, scope)`, use `entry.value` or `binding.default`.
   - For `"scan"`: call `connector.search({}, scope)`, filter by prefix, strip prefix from keys, apply `max_items`.
   - Wrap connector errors in `MemoryBindingResolutionError`.
