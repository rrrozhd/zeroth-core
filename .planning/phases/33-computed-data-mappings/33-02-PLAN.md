    case ast.Call():
        func = self._visit(node.func)
        if func not in _SAFE_BUILTIN_MAP.values():
            # Determine function name for error message
            if isinstance(node.func, ast.Name):
                func_name = node.func.id

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Graph author -> Expression engine | User-authored expressions with function calls cross into AST evaluation |
| Runtime namespace -> Expression engine | Run state, variables, visit counts are accessible to expressions |
| Orchestrator -> Mapping executor | Runtime context with run metadata flows into expression evaluation namespace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-06 | Tampering | _SafeEvaluator ast.Call handler | mitigate | Only callables present in `_SAFE_BUILTIN_MAP.values()` are executed. The check `func not in _SAFE_BUILTIN_MAP.values()` uses identity comparison against the actual Python builtin objects. Namespace poisoning cannot override builtins because `ast.Name` resolves safe builtins BEFORE namespace lookup. |
| T-33-07 | Information Disclosure | Context namespace exposure | accept | The context namespace exposes payload, state, variables, visit counts, path, and metadata. This is the intended design -- transform expressions need this data. No credentials, secrets, or internal Python state is included. The `metadata` field contains only `run_id`. |
| T-33-08 | Elevation of Privilege | Safe builtin escape via sorted(key=...) | mitigate | The `sorted` builtin accepts a `key` keyword argument. However, the key value would need to be a callable, and the only callables resolvable in the namespace are safe builtins themselves. Passing `key=len` works and is safe. Passing an arbitrary lambda is impossible because lambda is not in the AST allowlist. |
| T-33-09 | Denial of Service | Expensive safe builtin calls | accept | `sorted()` on very large lists could be slow. However, payload sizes are bounded by the platform's contract validation and the upstream node's output size. This is a low-risk scenario bounded by existing platform constraints. |
</threat_model>
