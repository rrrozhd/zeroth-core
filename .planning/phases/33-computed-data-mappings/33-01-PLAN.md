    ```python
    def _validate_expression(self, expression: str) -> None:
        """Check that a transform expression is syntactically valid and uses only safe AST nodes."""
        if not expression or not expression.strip():
            raise MappingValidationError("transform expression must not be empty")
        try:
            tree = ast.parse(expression, mode="eval")
        except SyntaxError as exc:
            raise MappingValidationError(
                f"invalid transform expression syntax: {expression!r}"
            ) from exc
        self._check_ast_safety(tree.body, expression)
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Graph author -> Expression engine | User-authored expression strings cross into AST evaluation |
| Source payload -> Transform namespace | Untrusted node output data becomes part of the evaluation namespace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | _SafeEvaluator expression input | mitigate | AST allowlist in `_SafeEvaluator._visit` structurally blocks all unsupported node types including ast.Call, ast.Import, ast.FunctionDef. Static validation in `MappingValidator._check_ast_safety` catches unsafe expressions at graph definition time before they reach runtime. |
| T-33-02 | Information Disclosure | Expression namespace access | mitigate | Namespace is constructed from a fixed set of keys (payload, state, variables, node_visit_counts, edge_visit_counts, path, metadata). No access to Python globals, builtins, os, sys, or any module. `_SafeEvaluator` resolves names only from the provided namespace dict. |
| T-33-03 | Elevation of Privilege | Expression code execution | mitigate | `_SafeEvaluator` has no `ast.Call` handler (function calls structurally blocked), no `ast.Import` handler (imports blocked), no dunder attribute traversal (attribute access returns None for non-Mapping types via getattr with default). Expression evaluation is purely data-in/data-out. |
| T-33-04 | Denial of Service | Expression complexity/loops | accept | Expressions are single-line eval-mode AST trees -- no loops, no recursion, no generators. Complex nested expressions are bounded by AST depth. Python's own parser limits handle pathological input. Risk is low for single expressions. |
| T-33-05 | Spoofing | MappingExecutionError wrapping | accept | ConditionEvaluationError is re-raised as MappingExecutionError with the original message. No credential data flows through error messages. Exception chaining (`from exc`) preserves the full traceback for debugging. |
</threat_model>
