<objective>
Wire ParallelExecutor into the RuntimeOrchestrator's _drive() loop, implementing per-branch governance (audit, policy, contract validation), budget pre-reservation, and global step tracking. The fan-out detection is a single conditional in _drive() after _dispatch_node() returns. Each branch reuses existing _dispatch_node(), _record_history(), and _enforce_policy() with branch-isolated state.

Purpose: This completes Phase 38 by connecting the parallel execution engine (Plan 01) to the production orchestrator with full governance. After this plan, graph authors can configure parallel fan-out/fan-in on any agent or executable unit node.

Output: Modified `src/zeroth/core/orchestrator/runtime.py`, enhanced `src/zeroth/core/parallel/executor.py`, and integration tests in `tests/parallel/`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Branch coroutines -> shared Run state | Concurrent branches must not mutate parent Run (isolation boundary) |
| External budget service -> BudgetEnforcer | Fail-open when unreachable; untrusted latency/availability |
| Branch output -> fan-in merge | Untrusted branch outputs collected into aggregated payload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-07 | T (Tampering) | Run state during parallel execution | mitigate | BranchContext provides isolated deep copies of all mutable Run fields; branches never receive a reference to the parent Run's mutable dicts/lists; merge happens sequentially after barrier |
| T-38-08 | D (DoS) | Budget pre-reservation | mitigate | check_budget() called before spawning; fail-open design when BudgetEnforcer is None or unreachable (consistent with existing D-12 pattern) |
| T-38-09 | R (Repudiation) | Per-branch audit | mitigate | Each branch produces its own NodeAuditRecord with branch_id and branch_index in execution_metadata, linked to parent run_id; audit records are immutable once written |
| T-38-10 | E (Elevation) | Policy bypass via parallel execution | mitigate | _enforce_policy() called per branch before each downstream node dispatch; policy denial in one branch does not bypass policy in others |
| T-38-11 | D (DoS) | Orphaned tasks on fail-fast | mitigate | Explicit asyncio.Task cancellation in fail-fast mode (from Plan 01 executor); await cancellation before returning to prevent resource leaks |
| T-38-12 | I (Info Disclosure) | Branch output leakage | accept | All branches belong to the same run/tenant; cross-branch data visibility within the same run is expected behavior |
</threat_model>
