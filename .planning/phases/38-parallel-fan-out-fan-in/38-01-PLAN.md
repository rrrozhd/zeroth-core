## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Graph author -> ParallelConfig | Untrusted configuration specifying split_path, max_branches, fail_mode |
| Node output -> split_fan_out | Untrusted output data traversed via dot-path for fan-out splitting |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | T (Tampering) | ParallelConfig | mitigate | Pydantic extra="forbid" + Literal types for merge_strategy/fail_mode prevent injection of unexpected fields/values |
| T-38-02 | D (DoS) | split_fan_out | mitigate | max_branches field caps branch count; FanOutValidationError raised if exceeded; empty list rejected |
| T-38-03 | D (DoS) | GlobalStepTracker | mitigate | asyncio.Lock-protected counter enforces max_total_steps across all branches, preventing runaway parallel execution |
| T-38-04 | T (Tampering) | BranchContext state | mitigate | BranchContext uses isolated copies (empty dicts/lists per D-05); branches never share mutable Run state |
| T-38-05 | I (Info Disclosure) | split_path traversal | accept | _get_path only reads from output_data dict (no filesystem/network access); traversal limited to dict keys, no dunder access |
| T-38-06 | D (DoS) | execute_branches fail-fast | mitigate | Explicit task cancellation in fail-fast mode prevents orphaned background tasks consuming resources |
</threat_model>
