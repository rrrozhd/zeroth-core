<objective>
Wire the artifact store into the orchestrator (TTL refresh on checkpoints), audit system (evidence export with optional payload resolution), contract registry (ArtifactReference structural validation), and service bootstrap. Create integration tests proving the full lifecycle works end-to-end.

Purpose: Completes the artifact store feature by connecting the core package (Plan 01) to the rest of the platform. Without this wiring, artifacts would be storable but their TTLs would expire during long approval waits, audit exports couldn't retrieve full payloads, and contracts couldn't validate artifact reference structures.

Output: Fully integrated artifact store with orchestrator TTL refresh, audit evidence resolution, contract validation support, bootstrap wiring, and comprehensive integration tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Orchestrator -> ArtifactStore | Orchestrator calls refresh_ttl with keys extracted from serialized run state |
| Audit export -> ArtifactStore | Evidence export resolves artifact references; base64 encoding prevents injection |
| Bootstrap -> Store construction | Settings determine which backend is instantiated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-06 | Information Disclosure | evidence.py resolve_artifact_references | mitigate | Only resolve when explicitly requested (resolve_artifacts=True); base64-encode raw bytes to prevent injection; never auto-resolve in default audit output |
| T-34-07 | Denial of Service | runtime.py _refresh_artifact_ttls | mitigate | Wrap in try/except to never fail a run due to TTL refresh; bound the number of refs scanned per run (log warning if > 1000) |
| T-34-08 | Spoofing | helpers.py extract_artifact_refs duck-typing | accept | Duck-typing may match non-artifact dicts that happen to have store/key/content_type/size fields; model_validate catches invalid structures; worst case is a harmless TTL refresh attempt on a non-artifact key |
| T-34-09 | Tampering | bootstrap.py store construction | mitigate | Only "filesystem" and "redis" backends accepted; unknown backend value raises ValueError at startup rather than silently falling back |
</threat_model>
