    **ArtifactStore Protocol:** Per D-01, define as `typing.Protocol` with six async methods:
    - `async def store(self, key: str, data: bytes, content_type: str, ttl: int | None = None) -> ArtifactReference`
    - `async def retrieve(self, key: str) -> bytes`
    - `async def delete(self, key: str) -> bool`
    - `async def refresh_ttl(self, key: str, ttl: int) -> bool`
    - `async def exists(self, key: str) -> bool`
    - `async def cleanup_run(self, run_id: str) -> int` (per D-09)

    **RedisArtifactStore:** Per D-02 and RESEARCH.md Pattern 2:
    - Constructor takes `redis_url: str`, `prefix: str = "zeroth:artifact"`, `default_ttl: int = 3600`, `max_size: int = 104857600`.
    - Creates `redis.asyncio.Redis.from_url(redis_url)` client.
    - `store()`: Validate size against max_size. Build full_key = `{prefix}:{key}`, meta_key = `{full_key}:meta`. Use pipeline(transaction=True) with setex (if TTL) or set (no TTL) for both keys. Return ArtifactReference.
    - `retrieve()`: Get full_key. If None, raise ArtifactNotFoundError. Return bytes.
    - `delete()`: Pipeline delete of both data key and meta key. Return bool based on delete count > 0.
    - `refresh_ttl()`: Check exists first. If not exists, raise ArtifactTTLError. Pipeline expire on both keys. Return True.
    - `exists()`: Return bool(await self._client.exists(full_key)).
    - `cleanup_run()`: Use scan_iter(match=f"{prefix}:{run_id}/*", count=100). Delete each key. Return count.

    **FilesystemArtifactStore:** Per D-02 and RESEARCH.md Pattern 3:
    - Constructor takes `base_dir: str | Path`, `default_ttl: int = 3600`, `max_size: int = 104857600`.
    - All blocking I/O wrapped in `asyncio.to_thread()`.
    - `store()`: Validate size. Validate key has no `..` segments (path traversal prevention per RESEARCH security domain). Write file at `{base_dir}/{key}`. Write sidecar at `{base_dir}/{key}.meta.json` containing content_type, size, created_at ISO, ttl_seconds, expires_at ISO (or None).
    - `retrieve()`: Read file. If not found, raise ArtifactNotFoundError. Check sidecar for expiration. If expired, delete both files and raise ArtifactNotFoundError (lazy cleanup per RESEARCH discretion).
    - `delete()`: Remove file and sidecar. Return True if file existed, False otherwise.
    - `refresh_ttl()`: Read sidecar. If not found, raise ArtifactTTLError. Update expires_at. Write sidecar.
    - `exists()`: Check file exists AND not expired per sidecar.
    - `cleanup_run()`: Delete entire `{base_dir}/{run_id}/` directory tree using shutil.rmtree wrapped in to_thread. Return count of deleted files.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Node output -> ArtifactStore | Node-produced data crosses into storage backend; key is server-generated, never client-supplied |
| Filesystem path resolution | Storage key is used to construct file paths on disk |
| Redis key namespace | Artifact keys share Redis instance with other subsystems |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | Tampering | FilesystemArtifactStore path resolution | mitigate | Reject keys containing ".." segments in store/retrieve/delete; validate resolved path is within base_dir |
| T-34-02 | Denial of Service | ArtifactStore.store() | mitigate | Enforce max_artifact_size_bytes setting; reject payloads exceeding limit before writing to backend |
| T-34-03 | Spoofing | ArtifactReference key field | mitigate | ArtifactReference keys are generated server-side via generate_artifact_key(); never accept client-supplied keys in node output path |
| T-34-04 | Information Disclosure | Filesystem store sidecar metadata | accept | Sidecar contains content_type and size only (no payload data); if filesystem is readable by attacker, security boundary is already breached |
| T-34-05 | Tampering | Redis key namespace collision | mitigate | All Redis artifact keys prefixed with configurable prefix (default: "zeroth:artifact:"); keys namespaced by run_id to prevent cross-run access |
</threat_model>
