## Spacing Scale

Declared values (Tailwind v4 default + Studio conventions, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip gaps, icon-label gaps, port label inset |
| sm | 8px | Compact element spacing, badge padding, inline chip gaps |
| md | 16px | Default form field spacing, config card section gaps, modal body padding |
| lg | 24px | Inspector tab content padding, inputs modal padding |
| xl | 32px | Settings page section breaks, Prompts/Connectors page gutters |
| 2xl | 48px | Page-level vertical rhythm between major sections |
| 3xl | 64px | Empty state vertical centering offset |

**Exceptions for Phase 24:**
- **Status badge chip**: 20px minimum height (below `sm` baseline) — intentional to fit over node corners without obscuring content. Inline padding `2px 6px` (sub-token) for chip density.
- **Agent node post-run chips row**: 4px gap (xs) between chips; information density requirement from D-12 (four chips: status, tokens+cost, model, latency).
- **Tool port handle**: 12px diameter (matches existing Phase 22 data port handles, not a new size).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 600 (semibold) | 1.4 |
| Heading | 18px | 600 (semibold) | 1.3 |
| Code/Mono | 13px | 400 (regular) | 1.5 |

**Rationale:**
- Studio is a dense authoring surface (canvas + inspector + rails), so body is 14px not 16px — matches Phase 22/23 established size. Consistent with n8n-style canvas tools.
- Two weights only: 400 regular and 600 semibold. No 500 medium, no 700 bold.
- Code/Mono is a fourth size (13px) exclusively for CodeMirror prompt editor and JSON output rendering in the Results tab (D-14). Uses system mono stack (`ui-monospace`, `SFMono-Regular`, etc.). This is the only place mono is permitted.
- Heading 18px used for: config card section titles, Results tab step titles, Settings page tab labels, Connectors page subsection headers. No h1 larger than this — Studio is a single-app viewport, not a marketing site.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `#F8FAFC` | Canvas background, inspector surface, modal body, page background |
| Secondary (30%) | `rgba(118, 182, 205, 0.08)` on `#FFFFFF` | Nodes, cards, sidebar rails, config card panels, result step rows |
| Accent (10%) | `--color-studio-accent` `rgba(79, 205, 255, 1)` (cyan) | Selected node ring, primary CTA backgrounds, run status `running` badge, active tab underline, focused input ring |
| Destructive | `--color-studio-destructive` `rgba(255, 80, 80, 1)` | Stop button, failed node status badge, validation errors, destructive confirmation buttons |

**Accent reserved EXCLUSIVELY for:**
1. Primary CTA backgrounds: **Run** button (before run starts), **Save** button (workflow dirty state), **Create/Save** buttons in Settings/Prompts/Connectors pages
2. Selected node ring (Phase 22/23 existing pattern — retained)
3. Active tab underline in inspector (Properties | Results)
4. Focused form input ring (CodeMirror and `InspectorField` focus state)
5. `running` status badge background (pulsing)
6. Active navigation item in the left rail (Workflows, Prompts, Connectors, Settings)

**Accent is NEVER used for:**
- Link text within body copy (use `--color-studio-text` with underline on hover)
- Secondary buttons (use border + text only, no fill)
- Inactive tab labels
- Chip backgrounds for non-accent states (tokens, cost, model, latency chips are all secondary — neutral tint)
- Hover-only states (use `--color-studio-hover` instead)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (idle) | **Run workflow** |
| Primary CTA (running) | **Stop run** (red destructive button, replaces Run per D-07) |
| Secondary CTA (Settings) | **Save changes** |
| Add-new CTAs | **New prompt**, **New model**, **New MCP server**, **New execution unit**, **New env var** |

### Empty states

| Surface | Heading | Body |
|---------|---------|------|
| Prompts page (no prompts) | **No prompts yet** | Prompts are reusable system and user messages for your agents. Create your first prompt to use it in any agent node. [New prompt] |
| Connectors → MCP tab | **No MCP servers registered** | Register a Model Context Protocol server to give agents access to tools. [New MCP server] |
| Connectors → Execution Units | **No execution units registered** | Execution units run sandboxed code as tools. Register one to attach it to an agent. [New execution unit] |
| Connectors → Memory | **No memory connectors registered** | Memory connectors give agents long-term semantic recall. Register one to attach it to a Memory Resource node. [New memory connector] |
| Settings → Models | **No models registered** | Agents need a model to run. Register at least one model from your provider of choice. [New model] |
| Settings → Env vars | **No environment variables** | Store API keys and secrets here, then reference them by name in model and connector configs. [New env var] |
| Inspector → Results tab (no run yet) | **No results yet** | Run the workflow to see inputs, outputs, tokens, and cost for this node. |
| Inspector → Results tab (node not reached) | **Node did not run** | This node was skipped because an upstream node failed. Fix the upstream error and re-run. |
| Agent config → Variable bindings (no prompt selected) | **Select a prompt to bind variables** | Once a prompt is selected, any `{{variable}}` placeholders will appear here for binding. |

### Error states

| Scenario | Copy |
|----------|------|
| WebSocket disconnect | **Connection lost** — Reconnecting… (auto-retry with exponential backoff per D-03) |
| WebSocket reconnect failed (3+ attempts) | **Can't reach the server** — Check your network and refresh the page. |
| Run failed — provider error | **Run failed: {ProviderError}** — {message}. Check the API key for `{env_var_name}` in Settings → Env vars. |
| Run failed — missing env var | **Missing environment variable** — `{VAR_NAME}` is referenced but not set. Add it in Settings → Env vars. |
| Run failed — validation | **Workflow has errors** — Fix the red-highlighted nodes and try again. |
| Agent config — unbound variable | **Unbound variable: `{{var}}`** — Bind it to an upstream node output or provide a static value. |
| Model test connection failed | **Can't connect** — {error message}. Verify the base URL and API key env var. |
| MCP server connection failed | **Server unreachable** — {transport error}. Check the command or URL and try again. |
| Prompt save — duplicate name | **A prompt named "{name}" already exists** — Choose a different name or edit the existing one. |

### Destructive confirmations

All destructive actions use a modal confirmation with a red primary button. Pattern: `{Action name}: {confirmation copy}`.

| Action | Confirmation copy |
|--------|-------------------|
| Delete prompt | **Delete prompt "{name}"?** This prompt is used by {N} agent node(s). Deleting will break those nodes until you reassign. [Cancel] [Delete] |
| Delete model | **Delete model "{label}"?** {N} agent node(s) currently use this model and will need to be reassigned. [Cancel] [Delete] |
| Delete MCP server | **Remove MCP server "{name}"?** Agents using tools from this server will lose access on next run. [Cancel] [Remove] |
| Delete execution unit | **Delete execution unit "{name}"?** Agents attached to this unit will lose it on next run. [Cancel] [Delete] |
| Delete memory connector | **Delete memory connector "{name}"?** Agents with this memory will lose recall on next run. [Cancel] [Delete] |
| Delete env var | **Delete `{NAME}`?** The underlying secret will be removed from the vault. Any models or connectors referencing this variable will fail on next run. [Cancel] [Delete] |
| Stop running workflow | **Stop this run?** Any nodes still running will be cancelled. Completed node results are preserved. [Keep running] [Stop run] |
| Overwrite prompt version | **Save as new version?** The current version stays in history. Agents referencing "latest" will pick up the new version on next run. [Cancel] [Save new version] |

### Tone rules
- Second person ("you", "your"), never "the user"
- Active voice: "Register a model" not "A model can be registered"
- No ellipsis on buttons except for in-progress states ("Reconnecting…", "Testing…")
- Error messages always pair **what went wrong** with **what to do next**
- Never blame the user ("You entered the wrong key" → "Can't connect — verify the API key")
- Numbers under ten spelled out in body copy, digits in chips and badges (token counts, latency, cost)

## Interaction Contracts

### Run lifecycle
1. **Idle:** Run button cyan (accent), label "Run workflow". Clicking opens Inputs modal.
2. **Inputs modal:** Auto-generated form from Start node schema. Submit triggers `POST /runs`.
3. **Starting:** Button disabled briefly, spinner inline, label "Starting…".
4. **Running:** Button becomes red (destructive), label "Stop run". Node status badges animate from `pending` → `running` (cyan, pulsing) → `complete` (green check) or `failed` (red X).
5. **Complete:** Button returns to cyan "Run workflow". Node chips persist showing tokens/cost/model/latency.
6. **Failed:** First failed node turns red; downstream nodes grey out as `skipped`. Inspector auto-switches to Results tab on the failed node if selected.

### WebSocket connection state
- Shown as a small indicator dot in `CanvasControls.vue` near the Run button:
  - Green dot = connected
  - Amber dot pulsing = reconnecting
  - Red dot = disconnected (3+ failed reconnects)
- Hover shows the current state as a tooltip.

### Status badge animation
- `running` state pulses at 1.5s cycle (opacity 0.6 → 1.0). All other states are static.
- Transition from `running` → `complete` or `failed` is a 200ms cross-fade, no bounce, no scale.

### Tool edges
- Dashed stroke (dasharray `6 4`), purple (`--color-studio-edge-tool`)
- NOT user-drawable — rendered from agent config `tools` field. Read-only on canvas.
- Hover highlights the source agent and target tool chip.

### Typed port tooltips (D-50)
- 300ms hover delay before tooltip appears
- Content: port label (bold), port type (faint), expected schema (mono if present), last-run observed value (mono, truncated to 200 chars with expand-on-click)

### CodeMirror prompt editor
- Height: min 240px, max 480px (vertical resize handle at bottom)
- Theme matches Studio colors (light mode only for v2.0 — dark mode deferred)
- Jinja `{{variable}}` placeholders highlighted in accent cyan on a faint accent-muted background
- Save as new version is explicit (D-26, D-30); no autosave

### Keyboard
- Existing Phase 23 shortcuts preserved (no conflicts introduced)
- Inputs modal: Enter submits, Esc cancels
- Destructive confirmation modals: Esc cancels, Enter defaults to Cancel (not destructive button — safety)
