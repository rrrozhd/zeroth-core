**Part A: Refactor apps/studio/src/stores/canvas.ts with command pattern (per D-09, D-10, D-12)**

Keep all existing exports (CanvasNode, CanvasEdge interfaces, useCanvasStore name). Refactor internals:

1. Add CanvasCommand interface at top of file:
```typescript
export interface CanvasCommand {
  execute(): void
  undo(): void
  description: string
}
```

2. Add history state inside the store:
```typescript
const history = ref<CanvasCommand[]>([])
const historyIndex = ref(-1)
const MAX_HISTORY = 50
const isExecutingCommand = ref(false)
```

3. Add `executeCommand` function:
- Truncate any redo history: `history.value = history.value.slice(0, historyIndex.value + 1)`
- Call `command.execute()`
- Push command to history
- If history exceeds MAX_HISTORY, shift oldest entry, else increment historyIndex

4. Add `undo()` function: if historyIndex >= 0, call `history.value[historyIndex.value].undo()`, decrement historyIndex.

5. Add `redo()` function: if historyIndex < history.value.length - 1, increment historyIndex, call `history.value[historyIndex.value].execute()`.

6. Add computed getters:
```typescript
const canUndo = computed(() => historyIndex.value >= 0)
const canRedo = computed(() => historyIndex.value < history.value.length - 1)
```

7. Wrap existing mutations in command factories. Keep the existing `addNode`, `removeNode`, `addEdge`, `removeEdge` for backward compatibility (they still work without undo). Add new command-based versions:

- `addNodeWithUndo(type: string, position: XYPosition)`: creates an AddNode command that on execute() creates the node (using existing addNode logic) and stores the created id; on undo() removes that node and its connected edges.

- `removeNodeWithUndo(id: string)`: creates a RemoveNode command that on execute() snapshots the node and its connected edges BEFORE removal, then removes; on undo() restores the node and its edges.

- `addEdgeWithUndo(connection: Connection)`: creates an AddEdge command; on execute() adds edge and stores id; on undo() removes that edge.

- `removeEdgeWithUndo(id: string)`: creates a RemoveEdge command; on execute() snapshots edge then removes; on undo() restores edge.

- `moveNodeWithUndo(id: string, from: XYPosition, to: XYPosition)`: creates a MoveNode command; on execute() sets node position to `to`; on undo() sets to `from`.

- `updateNodePropertyWithUndo(nodeId: string, key: string, oldValue: unknown, newValue: unknown)`: creates an UpdateProperty command; on execute() sets `node.data[key] = newValue`; on undo() sets `node.data[key] = oldValue`.

- `deleteSelected()`: gets all nodes where selected flag is true (or use selectedNodeId from uiStore), removes each with undo support. For simplicity, create a compound command that bundles multiple removals.

- `selectAll()`: selects all nodes (sets selected = true on each node).

- `copySelected()` / `pasteClipboard()` / `duplicateSelected()`: maintain an internal `clipboard` ref of serialized nodes+edges. Copy stores selected nodes and inter-edges. Paste creates new nodes with new IDs at 24px offset. Duplicate = copy + immediate paste.

- `clearHistory()`: resets history array and index.

8. Export all new functions and computed properties: `undo, redo, canUndo, canRedo, executeCommand, addNodeWithUndo, removeNodeWithUndo, addEdgeWithUndo, removeEdgeWithUndo, moveNodeWithUndo, updateNodePropertyWithUndo, deleteSelected, selectAll, copySelected, pasteClipboard, duplicateSelected, clearHistory, isExecutingCommand`
