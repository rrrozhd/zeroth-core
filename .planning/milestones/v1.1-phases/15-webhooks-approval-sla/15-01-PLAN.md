    ```python
    class WebhookRepository:
        def __init__(self, database: AsyncDatabase):
            self._database = database

        # Subscription CRUD
        async def create_subscription(self, sub: WebhookSubscription) -> WebhookSubscription: ...
        async def get_subscription(self, subscription_id: str) -> WebhookSubscription | None: ...
        async def list_subscriptions(self, deployment_ref: str | None = None, tenant_id: str | None = None) -> list[WebhookSubscription]: ...
        async def list_subscriptions_for_event(self, deployment_ref: str, event_type: WebhookEventType) -> list[WebhookSubscription]: ...
        async def deactivate_subscription(self, subscription_id: str) -> None: ...
        async def delete_subscription(self, subscription_id: str) -> None: ...

        # Delivery lifecycle
        async def enqueue_delivery(self, delivery: WebhookDelivery) -> WebhookDelivery: ...
        async def claim_pending_delivery(self) -> WebhookDelivery | None: ...
        async def mark_delivered(self, delivery_id: str) -> None: ...
        async def mark_failed(self, delivery_id: str, *, error: str, status_code: int | None, retry_delay: float) -> None: ...
        async def dead_letter(self, delivery_id: str) -> None: ...

        # Dead-letter queries
        async def list_dead_letters(self, subscription_id: str | None = None, limit: int = 50) -> list[WebhookDeadLetter]: ...
        async def get_dead_letter(self, dead_letter_id: str) -> WebhookDeadLetter | None: ...
    ```

    For `claim_pending_delivery`: Use atomic SELECT + UPDATE pattern. First SELECT delivery_id WHERE status='pending' AND next_attempt_at <= now() ORDER BY next_attempt_at LIMIT 1, then UPDATE status to 'delivering' (or use a two-step: fetch + immediate update in same transaction). Single-worker model per research recommendation, so simple SELECT-then-UPDATE within a transaction is sufficient.

    For `mark_failed`: Compute `next_attempt_at` using exponential backoff: `delay = min(retry_base_delay * 2^attempt_count, retry_max_delay)` with full jitter `random.uniform(0, delay)`. Update attempt_count += 1, next_attempt_at, last_error, last_status_code, status = 'failed' if attempt_count < max_attempts.

    For `dead_letter`: INSERT into webhook_dead_letters from the delivery data, then UPDATE delivery status to 'dead_letter'.

    Store subscription event_types as JSON array string using `to_json_value(sub.event_types)` and deserialize with `load_typed_value(row["event_types"], list)`.
