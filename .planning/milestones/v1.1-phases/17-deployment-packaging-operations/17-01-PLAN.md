    **1. Create `src/zeroth/service/health.py`:**

    Define Pydantic models (per D-13):
    ```python
    class DependencyStatus(BaseModel):
        model_config = ConfigDict(extra="forbid")
        status: str  # "ok" | "unavailable" | "error"
        latency_ms: float | None = None
        detail: str | None = None

    class ReadinessResponse(BaseModel):
        model_config = ConfigDict(extra="forbid")
        status: str  # "ok" | "degraded" | "unhealthy"
        checks: dict[str, DependencyStatus] = Field(default_factory=dict)

    class LivenessResponse(BaseModel):
        model_config = ConfigDict(extra="forbid")
        status: str = "ok"
    ```

    Implement async check functions:
    - `check_database(db: AsyncDatabase) -> DependencyStatus`: Execute `SELECT 1` via `db.transaction()`, measure latency with `time.monotonic()`, catch Exception -> status="error" with detail=str(exc).
    - `check_redis(redis_settings: RedisSettings) -> DependencyStatus`: Create `redis.asyncio.Redis` from settings, call `ping()`, measure latency. Catch Exception -> status="error".
    - `check_regulus(base_url: str | None, timeout: float = 5.0) -> DependencyStatus`: If base_url is None, return status="unavailable". Otherwise `httpx.AsyncClient.get(base_url + "/health")`, measure latency. Catch Exception -> status="unavailable".

    Implement `register_health_routes(app: FastAPI) -> None` (per D-07, D-10, D-11, D-12):
    - `@app.get("/health/ready", response_model=ReadinessResponse)` -- runs all three checks concurrently via `asyncio.gather`, determines overall status: "unhealthy" if DB or Redis failed, "degraded" if only Regulus failed, "ok" otherwise. Return HTTP 200 always (status in body, per convention).
    - `@app.get("/health/live", response_model=LivenessResponse)` -- return `LivenessResponse()` immediately.
    - Access database via `request.app.state.bootstrap` -- the bootstrap holds the database implicitly through repositories. For the health check, store the `AsyncDatabase` instance on `app.state` during `create_app`. Add `app.state.database = None` initially, and set it in bootstrap wiring.

    Actually, looking at bootstrap.py, the `AsyncDatabase` is passed to `bootstrap_service()` but not stored on ServiceBootstrap. The health check needs direct DB access. The cleanest approach: add an optional `database: AsyncDatabase | None = None` field to ServiceBootstrap dataclass, and set it during `bootstrap_service()`. Then health.py reads `app.state.bootstrap.database`.
